3 /*-------------------------------------------------------
4 Interfaz 2016, all rights reserved.
5 
6 This code serves as a bridge between the pattern-finder
7 algorithm implemented in python and the OPTITRACK SDK
8 coded in C / C++. For this purpose, both the Python C-API
9 and the numpy API are used.
10 
11 Besides some mandatory steps required for the APIs to work,
12 the code begins starting the cameras and importing the track
13 function from the tracker python module.
14 
15 Once the previous steps are completed an infinite loop is
16 started, where frames are retrieved, cast to python objects
17 and passed to the track function. The output is cast back
18 to a C++ array and the coordinates extracted from it are
19 accumulated till data from both cameras are retrieved. Then,
20 the stereo_triangulation function is called in order to get
21 the corresponding 3-D coordinates.
22 
23 Author: Jesús Muñoz
24 Version: 0.1
25 ---------------------------------------------------------*/
